## Overview

The `security` module provides enterprise-grade security features: PII detection and sanitization, audit logging, rate limiting, and configurable guardrails for AI outputs. Uses regex-based detection for zero-dependency, Edge-compatible operation.

**No peer dependencies required.**

## Quick Start

```ts
import { detectPII, sanitizeForLLM } from '@jamaalbuilds/ai-toolkit/security';

const findings = detectPII('Contact John Smith at john@example.com');
// [{ type: 'EMAIL', value: 'john@example.com', ... }, { type: 'NAME', ... }]

const safe = sanitizeForLLM('My SSN is 123-45-6789');
// 'My SSN is [REDACTED_SSN]'
```

## API Reference

### `detectPII(text)`

Detect personally identifiable information in text.

```ts
function detectPII(text: string): PIIFinding[]
```

Detects: `EMAIL`, `PHONE`, `SSN`, `CREDIT_CARD`, `NAME`, `DOB`, `IP_ADDRESS`, `URL`

```ts
const findings = detectPII('John Smith, born 1990-01-15, SSN 123-45-6789');
for (const f of findings) {
  console.log(`${f.type}: ${f.value} at position ${f.start}-${f.end}`);
}
```

### `sanitizeForLLM(text)`

Replace detected PII with redaction tokens before sending to an LLM.

```ts
function sanitizeForLLM(text: string): string
```

```ts
const safe = sanitizeForLLM('Email me at john@example.com');
// 'Email me at [REDACTED_EMAIL]'
```

### `createRateLimiter(config)`

Create a rate limiter for request throttling.

```ts
function createRateLimiter(config: RateLimitConfig): RateLimiter
```

| Parameter | Type | Description |
| --- | --- | --- |
| `config.maxRequests` | `number` | Maximum requests per window |
| `config.windowMs` | `number` | Window duration in milliseconds |

```ts
const limiter = createRateLimiter({ maxRequests: 100, windowMs: 60_000 });
const result = limiter.check('user-123');
if (!result.allowed) {
  console.log(`Rate limited. Retry after ${result.retryAfterMs}ms`);
}
```

### `createAuditLogger()`

Create an audit logger for tracking security-relevant events.

```ts
function createAuditLogger(): AuditLogger
```

```ts
const audit = createAuditLogger();
audit.log({
  action: 'pii_detected',
  userId: 'user-123',
  details: { types: ['EMAIL', 'SSN'], sanitized: true },
});
const events = audit.getEvents();
```

### `createGuardrails(rules)`

Create configurable guardrails for AI output validation.

```ts
function createGuardrails(rules: GuardrailRule[]): Guardrails
```

### `checkOutput(guardrails, text)`

Validate AI output against guardrail rules.

```ts
function checkOutput(guardrails: Guardrails, text: string): GuardrailResult
```

```ts
const guardrails = createGuardrails([
  { name: 'no-pii', check: (text) => detectPII(text).length === 0 },
  { name: 'max-length', check: (text) => text.length < 10000 },
]);

const result = checkOutput(guardrails, aiResponse);
if (!result.passed) {
  console.log('Failed rules:', result.failures);
}
```

## Types

- `PIIFinding` — type, value, start, end
- `PIIType` — `'EMAIL' | 'PHONE' | 'SSN' | 'CREDIT_CARD' | 'NAME' | 'DOB' | 'IP_ADDRESS' | 'URL'`
- `RateLimiter` — limiter with check() method
- `RateLimitResult` — allowed, remaining, retryAfterMs
- `AuditLogger` — logger with log(), getEvents()
- `AuditEvent` — action, userId, details, timestamp
- `Guardrails` — validator instance
- `GuardrailResult` — passed, failures
- `GuardrailRule` — name, check function
